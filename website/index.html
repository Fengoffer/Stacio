WORKFLOW

一次远程操作，从连接到解决。

Stacio 把 SSH、文件、传输、隧道和诊断放进同一个操作循环，减少在终端、Finder、脚本和笔记之间来回切换。

01

Connect

从保存会话或快速连接进入本地/远程终端。

02

Inspect

查看终端上下文、远程目录、日志和设备状态。

03

Transfer

通过 Stacio 内置 SCP 工作流上传、下载和跟踪传输。

04

Tunnel

创建并监控 SSH 隧道，避免后台连接失控。

05

Resolve

结合诊断信息和 AI 建议完成排查。

SECURITY

本地优先，敏感操作始终由用户确认。

Stacio 把远程工作流收进同一个 macOS 应用，但凭据、会话和敏感动作仍遵循本地优先与显式确认。

凭据交给 macOS Keychain，本地会话元数据不默认外传。
AI 建议只生成排查思路和命令卡片，敏感命令需要用户确认后执行。
诊断、日志和设备信息默认脱敏，减少复制粘贴排查时的泄露风险。
Windows / Linux 入口保持预留状态，不写成已开放安装包。

UPDATE NOTES

Stacio 0.13.3-Beta 更新说明

本次更新重点提升远程终端稳定性，并完善更新、反馈和许可管理体验。

更新亮点

  • 修复部分 macOS 27 设备在打开 SSH 终端时可能闪退的问题。
  • 改进 SSH 会话启动、重连和远程目录联动，连接过程更加稳定。
  • 完善 AI 服务商与模型管理，可集中配置兼容 OpenAI API 的服务和模型。
  • 新增应用内更新通道、手动检查更新、用户反馈和许可状态入口。
  • 优化设备看板、终端、文件面板、隧道和设置页面的 macOS 原生界面细节。

安装说明

  • 支持 macOS 14 及以上版本，当前安装包适用于 Apple Silicon Mac。
  • 首次打开未公证 Beta 包时，可能需要在 Finder 中右键 Stacio 并选择“打开”。
  • 已保存的会话、凭据引用、AI 配置、传输记录和应用设置会继续保留。
